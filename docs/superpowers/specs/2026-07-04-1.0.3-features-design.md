# Manuscrito 1.0.3 features

Design for a batch of seven additions. All changes are to the single
`src/main.odin` file. No new files, no new dependencies.

## Scope

1. Splash placeholder text mentions F1 help.
2. Page number shown on each page in page view.
3. Current page shown next to the word/page count.
4. A fixed hint centered in the status bar.
5. File name shown on the left of the status bar.
6. Five export formats: txt, rtf, doc, md, html.
7. Strikethrough character style, bound to Ctrl+J.

PDF export was considered and dropped. A from-scratch PDF writer is the single
largest piece of work here and is not needed for this release. It can be added
later without disturbing the export dispatch built below.

## 1. Splash text

In `draw_document`, the empty-document placeholder at line 1917 changes from
`"Start writing."` to `"Start writing. Press F1 for help."`.

## 2 and 5. Status bar

The status bar (the 52px strip at the bottom) is reorganized into three zones.

**Left, x=24:** the file name. When a file is open, show its basename
(`filepath.base`). When no file is open, show `New document`. Append a trailing
` •` when `app.dirty` is set. This replaces the old behavior where the left
showed the transient status and the word "Unsaved changes".

**Center:** chosen by priority each frame:
- If `app.confirm_pending`, show the confirm prompt (`Press Ctrl+Q again to
  quit.` or `Unsaved changes. Repeat the command to discard.`). This persists
  until the pending action resolves.
- Else if a transient message is fresh, show it. Freshness is
  `rl.GetTime() - app.status_time < 2.5`.
- Else show the fixed hint `Press Ctrl+P to open the Command Palette`.

**Right:** the count text, described in section 3.

To time transient messages, add a field to `App`:
- `status_time: f64`

Add a helper `set_status :: proc(app: ^App, msg: cstring)` that sets
`app.status = msg` and `app.status_time = rl.GetTime()`. Replace the direct
`app.status = ...` assignments that represent transient events (Saved, Copied,
Pasted, Opened, Found, Not found, Open failed, Save failed, New document, theme
name) with `set_status`. The confirm-prompt assignments in `execute_command`
stay as direct writes to `app.status` and are read only through the
`confirm_pending` branch, so they are not subject to the timeout.

The three zones are drawn independently. The center string is measured and
centered on the screen width. No collision handling between zones: this is a
fullscreen editor and the window is always wide.

## 3. Current page

The right zone shows `Page N · W words / P pages` where:
- `W` is `count_words(app)`.
- `P` is the existing page count: the laid-out page count in page view, or
  `(words + 249) / 250` otherwise.
- `N` is the cursor's current page. In page view,
  `floor(cursor_document_y(app, content_w, base_size) / period) + 1`. Off page
  view, `words_before_cursor / 250 + 1`, computed by counting words in
  `app.text[:app.cursor]`.

`cursor_document_y` and `page_metrics` already exist and are reused. When
`period` is 0 (page view off) the estimate branch is used.

## 4. Page number on the page

Inside `draw_document`'s page-rectangle loop (the `if period > 0` branch, around
line 1900), draw the page number for each visible page. Text is `k+1` in the UI
font at a small size in `theme.muted`, positioned at the bottom-right corner
inside the page rectangle, inset by a small margin. Only page view draws these;
the non-page background branch draws nothing.

## 7. Strikethrough

A new character style, parallel to underline in every respect.

- `Char_Style` gains `strike: bool`.
- `Style_Field` gains `Strike`.
- `get_style_field` and `set_style_field` handle `.Strike`.
- `same_style` compares `strike`.
- `style_code` uses bit `16` (`1 << 4`); `style_from_code` reads it. Existing
  saved files have the bit clear, so they load as not-struck. No format version
  bump needed.
- `execute_command` gains a `.Strike` command case calling
  `toggle_style(app, .Strike)`, plus a `Command_Kind.Strike` enum value and a
  `COMMANDS` entry `{"Strikethrough", .Strike}`.
- `handle_shortcuts` maps `Ctrl+J` to `execute_command(app, .Strike)`.
- `HELP_ENTRIES` gains a line, folded into the existing Bold/Italic/Underline
  entry or added next to it: `Ctrl+J  Strikethrough`.
- Rendering in `draw_visual_line`: a horizontal line through the glyph middle,
  drawn as continuous runs across struck characters, exactly like the existing
  underline run logic but at `y + font_size * 0.5` instead of below the
  baseline.

## 6. Export

Export reuses the existing path-prompt machinery rather than adding a file
dialog.

**Format enum:** `Export_Format :: enum int { Txt, Rtf, Doc, Md, Html }`.

**App state:** add `export_format: Export_Format`.

**Path action:** add `Export` to `Path_Action`.

**Commands:** add five `Command_Kind` values and `COMMANDS` entries:
`Export as TXT`, `Export as RTF`, `Export as DOC`, `Export as Markdown`,
`Export as HTML`. Each command case sets `app.export_format` and calls
`begin_path_prompt(app, .Export)`.

**Prompt prefill:** `begin_path_prompt` for `.Export` prefills `path_input` with
the current file's base name (or `document` when unsaved) plus the extension for
the chosen format (`.txt`, `.rtf`, `.doc`, `.md`, `.html`).

**Prompt title:** `draw_path_prompt` shows `Export as TXT` and so on.

**On Enter:** `update_path_prompt`'s `.Export` case calls
`export_document(app, path, app.export_format)`, sets a transient status
(`Exported` or `Export failed`), and closes the prompt.

**Dispatch:** `export_document` builds a string with a per-format serializer and
writes it with `os.write_entire_file`.

Content rules across formats. Text always. Where the format allows: headings
(the paragraph header level, mapped to that format's sizing), paragraph breaks,
first-line indent, italic, bold, underline, strikethrough, and paragraph
alignment. Never the theme. Highlight is not exported.

Per-format detail:

- **TXT:** the runes of each paragraph, paragraphs separated by a blank line. No
  styling, no headings markup (a heading is just its text).
- **Markdown:** headings as `#` to `####` by level; `**bold**`, `*italic*`,
  `~~strike~~`; underline as `<u>...</u>` since Markdown has no underline.
  Paragraphs separated by a blank line. First-line indent and alignment are not
  representable in plain Markdown and are omitted.
- **HTML:** a minimal complete document (`<!doctype html>` with a `<meta
  charset>`). Headings as `<h1>` to `<h4>`, body paragraphs as `<p>`. Inline
  styles as `<b> <i> <u> <s>`. First-line indent as inline
  `style="text-indent:2em"`, alignment as inline `style="text-align:..."`.
  Text is HTML-escaped (`& < >`).
- **RTF:** `{\rtf1\ansi\deff0 ... }`. Runs use `\b \i \ul \strike` toggles reset
  per run. Heading size via `\fs` (points doubled per RTF convention; base body
  size maps to a fixed point size, headings scale up by the same ratios as
  `header_size`). Each paragraph is `\pard` plus alignment (`\ql \qc \qr \qj`)
  plus `\fi720` when first-line indent is set, ending with `\par`. Escape `\`,
  `{`, `}`; emit non-ASCII runes as `\u<code>?`.
- **DOC:** byte-identical to the RTF output, written to the `.doc` path. Word and
  LibreOffice open RTF content under a `.doc` extension. A true binary `.doc` is
  infeasible to write from scratch and is not attempted.

A small shared helper walks paragraphs and their style runs (the same run
grouping `serialize_document` uses via `same_style`) so each serializer receives
runs rather than re-scanning characters.

## Testing

`src/wrap_test.odin` holds the existing numeric wrap tests. Add assert-based
checks for the pieces with real logic:
- `style_code` round-trips through `style_from_code` with the strike bit set.
- Each serializer on a small fixed document with one heading, one bold run, one
  italic run, one struck run, and a first-line-indented paragraph produces
  output containing the expected markers (for example RTF contains `\strike`,
  HTML contains `<s>`, Markdown contains `~~`).

The one-line and pure-drawing changes (splash text, page number placement,
status zones) need no automated test.
